# Docker Web 管理界面

## Problem Frame

Docker 容器运行时无法使用系统托盘（pystray），现有 GUI 模式需要 X11 转发，配置复杂且仅适用于 Linux 桌面环境。需要一种更通用的管理方式，让 Docker 用户能够：

- 查看服务状态并控制启停
- 查看日志
- 切换配置预设
- 修改运行参数

Web 管理界面是跨平台的标准解决方案，任何有浏览器的设备都可以访问。

## Requirements

**核心功能**

- R1. 服务状态显示：显示运行状态（运行中/已停止）、监听端口、运行时间
- R2. 服务控制：启动/停止服务按钮（停止前需确认对话框）
- R3. 日志查看：分页显示日志，支持按级别过滤（INFO/ERROR），自动刷新选项。注意：日志内容已过滤敏感头字段（Authorization、x-api-key）

**配置管理**

- R4. 预设管理：列出可用预设，一键切换（切换前需确认对话框），显示当前激活的预设
- R5. 配置编辑：修改代理监听端口（管理界面共用此端口，修改后显示新访问地址）、日志等级，保存后需重启容器生效

**访问与安全**

- R6. 访问方式：复用代理端口，管理界面路径为 `/_admin`（如 `http://ip:8087/_admin`），使用下划线前缀避免与代理路由冲突
- R7. 绑定地址：监听 `0.0.0.0`，支持局域网访问。注意：此配置会同时暴露代理 API（无认证）和管理界面（需密码）到局域网，用户应确保网络环境可信或通过反向代理添加 TLS
- R8. 密码保护：
  - 配置文件中以 bcrypt 哈希存储密码（字段 `admin_password_hash`）
  - 登录失败 5 次后锁定 15 分钟
  - 会话令牌有效期 24 小时，存储于内存中

**清理工作**

- R9. 移除 Docker GUI 模式：删除 `docker-compose.yml` 中的 `llm-route-gui` 服务、`Dockerfile` 中的 `full` stage、相关 X11 挂载配置

**交互状态**

- R10. 加载状态：初始页面加载、日志刷新、配置保存、服务启停时显示加载指示器
- R11. 错误处理：操作失败时显示错误消息，支持重试
- R12. 禁用状态：服务运行时禁用"启动"按钮，服务停止时禁用"停止"按钮

**无障碍访问**

- R13. 键盘导航：所有操作可通过键盘完成，Tab 键顺序合理
- R14. ARIA 标签：交互元素有明确的 ARIA 标签描述
- R15. 颜色对比：文本与背景对比度至少 4.5:1

## Success Criteria

- Docker 容器启动后，可通过 `http://<容器IP>:8087/_admin` 访问管理界面
- 密码验证正确后可使用全部管理功能：状态查看（R1）、服务控制（R2）、日志浏览（R3）、预设切换（R4）、配置编辑（R5）
- 登录失败 5 次后账户锁定 15 分钟
- 加载状态、错误提示、按钮禁用状态按 R10-R12 规范正确显示
- 无障碍访问符合 R13-R15 要求
- 原 GUI 模式相关代码和配置已完全移除
- Headless 模式继续正常工作，不受影响

## Scope Boundaries

- 不实现 Token 用量统计（后续可单独添加）
- 不实现用户系统，仅单密码保护
- 不修改桌面端的托盘功能
- 不实现 WebSocket 实时推送，日志采用轮询刷新
- 不实现 TLS，用户需自行配置反向代理（如 nginx/Traefik）或确保网络环境可信

## Key Decisions

- **复用代理端口**：管理界面作为代理服务器的子路径，无需额外端口，简化部署
- **路径前缀 `/_admin`**：下划线前缀表示内部路由，避免与代理 API 路径冲突
- **bcrypt 哈希存储**：密码不以明文存储，降低配置文件泄露风险
- **轮询而非 WebSocket**：日志刷新频率低，轮询实现简单，避免长连接复杂性
- **内存会话存储**：符合"简单密码保护"定位，服务重启后需重新登录

## Dependencies / Assumptions

- 假设代理服务器已有 `aiohttp` 框架，可直接添加 Web 路由
- 假设配置文件格式（YAML）支持添加 `admin_password_hash` 字段
- 假设用户理解绑定 0.0.0.0 的安全含义，或在可信网络环境中使用
- 需要添加 `bcrypt` 依赖用于密码哈希
- Docker 部署时配置文件应移除 `:ro` 标志以支持配置编辑
